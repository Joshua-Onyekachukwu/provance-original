# Decision Log Template

Use this template for decisions that affect architecture, security posture, data handling, pricing, legal claims, or long-lived product behavior.

## Decision

State the decision in one sentence.

## Status

One of:

- proposed
- approved
- rejected

## Date

YYYY-MM-DD

## Owner

One owner.

## Approvers

List required approvers, if any.

## Context

Short summary of why this decision is being considered.

## Options Considered

### Option A

- description
- pros
- cons
- risks

### Option B

- description
- pros
- cons
- risks

### Option C

- description
- pros
- cons
- risks

## Recommendation

State the recommended option and why it wins.

## Consequences

List the second-order effects and what becomes harder or easier.

## Risks And Mitigations

List risks and concrete mitigations.

## Rollout Plan

Describe the rollout sequence and rollback plan if applicable.

## Documentation Updates Required

List exact docs and files to update.

## Implementation Tasks

Link to tasks or PRs that implement the decision.
