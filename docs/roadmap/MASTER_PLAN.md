## 2. Product Strategy (The CPO Perspective)

### Product Vision
Provance is the **defensible trust layer** for the synthetic media era. We bridge the gap between "AI suspicion" and "Forensic proof."

### Growth Loops
1.  **The Report Flywheel:** Every PDF report exported by a journalist/lawyer contains a "Verified by Provance" watermark with a link back to the methodology page.
2.  **API Integration Loop:** Large platforms (e.g., social media, archives) integrate our API for batch verification, generating massive datasets for signal improvement.

### User Journey: The Investigation Workflow
1.  **Acquisition:** Professional encounters a Provance Forensic Report in a news article.
2.  **Education:** Interactive landing page teaches the "5 dimensions of forensic analysis."
3.  **Conversion:** Professional signs up for Pro tier to get their first 100 reports.
4.  **Action:** Media Upload -> Multi-Signal Analysis (Async) -> Verdict & Evidence Breakdown.
5.  **Artifact:** Download PDF -> Citations in news/court.

## 3. Technical Strategy (The CTO Perspective)

### System Architecture
- **Frontend:** React + Tailwind v4 + Framer Motion (Scroll-driven animations).
- **Core Engine (API):** Python (FastAPI) for forensic processing; Node.js (Next.js/Hono) for the workflow layer.
- **Data Persistence:** SQLite (Turso) for metadata; S3 (Encrypted) for media; Redis for job queues.

### Security Architecture
- **Privacy-First Processing:** Optional "Ephemeral Mode" where media is never written to disk, only processed in RAM.
- **Audit Logs:** Immutable action logs stored in a content-addressed ledger.
