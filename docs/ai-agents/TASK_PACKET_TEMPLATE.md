# Task Packet Template v1

Use this template for every non-trivial assignment.

## Objective

One sentence describing the outcome.

## Why This Matters

One short paragraph linking the task to product, customer, or company goals.

## Current State

Three to seven bullets describing what is already true.

## Source Of Truth

List the exact files and documents to use.

- [file path]
- [file path]

## Constraints

List only the constraints relevant to this task.

Examples:

- no em dashes in site copy
- backend-mediated auth remains current strategy
- do not modify legacy `api/` unless explicitly requested
- keep documentation in sync with implementation
- do not introduce new paid services without approval

## Acceptance Criteria

Three to seven bullets defining done.

## Non-Goals

Explicitly list what is out of scope.

## Risks

List meaningful risks and the exact escalation triggers.

## Owner

Exactly one owner agent.

## Collaborators

Only supporting agents required for delivery.

## Deliverable Format

Specify the exact output form expected:

- PR
- design spec
- architecture memo
- policy draft
- investor deck slide copy
- operational runbook

## Escalate If

List explicit escalation rules:

- security or data exposure risk
- irreversible migration risk
- legal or compliance ambiguity
- unclear user-facing claim boundaries
