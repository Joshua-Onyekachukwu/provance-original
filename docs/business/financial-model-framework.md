# Financial Model Framework

## Objective

Provide a decision framework for modeling growth, cost structure, pricing, and capital needs before full financial forecasting is finalized.

## Revenue Streams

- Pro subscription
- Team subscription
- Enterprise contracts
- Usage-based API revenue
- Paid forensic report exports
- Services / onboarding where justified

## Cost Structure

### Cost Of Goods Sold

- compute for inference
- object storage
- bandwidth
- queue and database usage
- third-party notifications or auth costs

### Operating Expenses

- salaries and contractors
- cloud platform base cost
- security and compliance tools
- legal and governance
- sales and marketing
- customer support

## Forecast Drivers

### Demand Drivers

- signups by segment
- activation rate
- paid conversion rate
- scans per active user
- enterprise contract count
- API request volume

### Cost Drivers

- average processing time by media type
- ratio of CPU to GPU jobs
- storage retention period
- average artifact size
- support cost per account

## Unit Economics

### Core Unit

Primary unit: cost and revenue per scan, per active account, and per team workspace.

### Key Metrics

- gross margin per scan
- gross margin per customer segment
- CAC by acquisition channel
- payback period
- LTV/CAC
- burn multiple

## Pricing Strategy

### Recommended Starting Structure

- Free Trial: limited scans and basic result access
- Pro: individual user plan with higher monthly scan allowance
- Team: shared workspace, report export, collaboration
- Enterprise: API, SSO, audit exports, support, custom volume pricing

### Pricing Principles

- price to decision value, not raw compute cost
- use volume controls to protect margins
- avoid unlimited plans in early product stages

## Break-Even Analysis Inputs

- monthly recurring revenue by segment
- cost per scan by media type
- fixed team cost
- infrastructure overhead
- customer acquisition cost

## Cash Flow Forecast Inputs

- runway start capital
- monthly burn
- hiring ramp
- infrastructure scale curve
- expected timing of first enterprise revenue

## Capital Strategy

### Pre-Seed Use Of Funds

- dataset and benchmark operations
- MVP build
- security and policy baseline
- pilot acquisition

### Funding Milestones

- pre-seed: prove MVP and first pilots
- seed: prove repeatable paid adoption and enterprise readiness

## Modeling Recommendations

1. Build base, conservative, and aggressive cases.
2. Split image and video costs separately.
3. Forecast enterprise sales conservatively.
4. Use gross margin thresholds as a gating metric for product expansion.
